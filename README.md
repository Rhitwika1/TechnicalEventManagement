# Technical Event Management System 

## Project Overview

Technical Event Management System (EMS) is a front-end web application developed using HTML, CSS, and JavaScript. The system manages Users, Vendors, and Admin through role-based login.

The project uses the browser's LocalStorage to store user and vendor data. No backend framework or database server is used.

---

## Technologies Used

- HTML  
- CSS  
- JavaScript  
- Browser LocalStorage  

---

## System Roles

The system supports three roles:

- User  
- Vendor  
- Admin  

Each role has a separate dashboard and different permissions.

---

## Features

### User Module

- User registration (Sign Up)  
- User login  
- User dashboard  
- Event planning tools  
- Guest list management  
- Browse vendor section  
- Logout functionality  

---

### Vendor Module

- Vendor registration  
- Vendor login  
- Vendor dashboard  
- Inventory management  
- Add and delete service items  
- Logout functionality  

---

### Admin Module

- Predefined admin login  
- View all registered users  
- View all registered vendors  
- Delete users  
- Delete vendors  
- System monitoring

## How Data is Stored

- User and vendor information is stored using LocalStorage.  
- Data remains saved in the browser until manually cleared.  
- No external database is used.
